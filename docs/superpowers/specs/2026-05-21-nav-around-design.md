# Дизайн: стрелки навигации вокруг календаря (`navLayout="around"`)

**Дата:** 2026-05-21
**Статус:** утверждён к реализации
**Базируется на:** библиотека `bp-mui-date-range-picker`

## Цель

Сейчас навигация (`nav` — пара стрелок ‹ ›) рендерится отдельным плавающим
элементом сверху-справа календаря. Нужно расположить стрелки так, чтобы:

- стрелка «назад» (‹) была слева от месяца и года первого календаря/месяца;
- стрелка «вперёд» (›) была справа от месяца и года последнего календаря/месяца.

## Подход

`react-day-picker` v9 поддерживает это встроенным пропом `navLayout="around"`.
Он рендерит `PreviousMonthButton` первым элементом **первого** месяца (перед
его `MonthCaption`) и `NextMonthButton` внутри **последнего** месяца (после его
`MonthCaption`), а отдельный `<Nav>` при заданном `navLayout` не рендерит вовсе.

Отклонённые альтернативы: двигать дефолтный плавающий `<nav>` через CSS
(хрупко — это один элемент с двумя кнопками, разнести их по краям костыльно);
переопределять компонент `MonthCaption` и вставлять кнопки вручную (изобретает
то, что `navLayout` даёт из коробки).

## 1. Поведение

`navLayout: 'around'` добавляется в общий объект пропов (`shared`) → действует в
обоих режимах:

- **contiguous** (один `DayPicker`, `numberOfMonths={2}`): «‹» рендерится в
  первом месяце (`displayIndex === 0`), «›» — в последнем
  (`displayIndex === numberOfMonths - 1`). Итог:
  ```
  ‹  [Месяц▾][Год▾]            [Месяц▾][Год▾]  ›
     Пн Вт Ср …                Пн Вт Ср …
     …сетка 1…                 …сетка 2…
  ```
- **non-contiguous** (два независимых `DayPicker`, по одному месяцу): в
  одномесячном пикере `displayIndex === 0` и `displayIndex === numberOfMonths-1`
  истинны одновременно, поэтому каждый календарь получает свою пару «‹ … ›»
  вокруг своей шапки. Календари навигируют независимо.

Отдельный плавающий `<nav>` (пара стрелок сверху-справа) больше не рендерится —
`react-day-picker` сам его убирает при заданном `navLayout`.

## 2. Стилизация

`rdpClassNames` заменяет дефолтные классы rdp (`rdp-month`,
`rdp-button_previous` и т.д.) собственными, поэтому встроенная CSS rdp для
`navLayout="around"` (завязана на `rdp-*` классы) не применяется.
Позиционирование задаётся собственными Tailwind-классами в слотах:

- **`month`** — добавляется `relative`: месяц становится контейнером
  позиционирования для абсолютно спозиционированных кнопок. Было `space-y-2` →
  `relative space-y-2`.
- **`caption`** — добавляются центрирование и горизонтальные отступы (≈ ширина
  кнопки навигации), чтобы стрелки по краям не перекрывали списки месяца/года.
- **`navButton`** — остаётся одним слотом для общего вида кнопки (размер,
  hover-состояние). Сторона задаётся в `rdpClassNames` через дополнительные
  аргументы `mergeSlot`: `button_previous` получает позиционирование к левому
  краю (`absolute left-0 top-0`), `button_next` — к правому
  (`absolute right-0 top-0`). Это внутренняя деталь, не настраивается
  потребителем — по аналогии с `caption_label: 'hidden'`.

## 3. Удаление мёртвого слота `nav`

При `navLayout="around"` отдельный `<nav>` не рендерится, поэтому слот `nav`
больше ничего не стилизует. Он удаляется из:

- union `Slot` в `src/types.ts`;
- `src/styles/defaultClassNames.ts`;
- маппинга в `rdpClassNames` (`src/components/RangeCalendar.tsx`).

Это обратно-несовместимое сужение типа `Slot` на одно значение; библиотека ещё
не опубликована, потребителей нет — удаление честнее, чем оставлять слот,
который ничего не стилизует.

## 4. Затрагиваемые файлы

- `src/types.ts` — убрать `'nav'` из union `Slot`.
- `src/styles/defaultClassNames.ts` — убрать запись `nav`; поправить `month`
  (добавить `relative`) и `caption` (центрирование + горизонтальные отступы).
- `src/components/RangeCalendar.tsx` — `navLayout: 'around'` в `shared`; в
  `rdpClassNames` убрать `nav`, у `button_previous`/`button_next` добавить
  позиционирование по сторонам.
- `src/components/RangeCalendar.test.tsx` — новые тесты.

## 5. Обработка ошибок

Особой обработки не требуется — это раскладочное изменение. Кнопки на границах
диапазона навигации (`startMonth`/`endMonth`) react-day-picker сам помечает
`aria-disabled`, как и прежде.

## 6. Тестирование (TDD)

- Отдельный элемент `<nav>` в DOM отсутствует (`navLayout` активен).
- Кнопки «назад» и «вперёд» рендерятся; в contiguous-режиме — ровно по одной.
- Порядок в DOM (contiguous): кнопка «назад» идёт перед шапкой первого месяца;
  кнопка «вперёд» — после шапки последнего месяца.
- Существующие тесты (количество сеток, клик по дню, выпадающие списки,
  скрытый `caption_label`) остаются зелёными.

## Объём

Одна небольшая фича, один план реализации. Изменение публичного API: union
`Slot` теряет значение `'nav'` (обратно-несовместимо, но потребителей нет).
