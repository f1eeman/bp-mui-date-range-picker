# Дизайн: дефолт `closeOnSelection` → `false`

**Дата:** 2026-05-21
**Статус:** утверждён к реализации
**Базируется на:** библиотека `bp-mui-date-range-picker`

## Цель

Сейчас при выборе полного диапазона (две разные даты) поповер с календарём
закрывается. Нужно, чтобы по умолчанию он **не закрывался**, а поведение
оставалось управляемым пропом.

## Контекст

Проп `closeOnSelection` уже существует и работает. `handleCalendarChange` в
`DateRangeInput` содержит `if (closeOnSelection && complete) setOpen(false)`.
Сейчас дефолт — `true`. Меняется только значение по умолчанию.

## Изменение

- **Поведение.** `closeOnSelection = false` по умолчанию: поповер не
  закрывается при выборе второй даты — пользователь закрывает его сам (Esc или
  клик вне). Потребитель, которому нужно прежнее поведение, передаёт
  `closeOnSelection` (`true`). Логика `handleCalendarChange` не меняется.
- **`src/DateRangeInput.tsx`** — в деструктуризации пропсов:
  `closeOnSelection = true` → `closeOnSelection = false`.
- **`src/types.ts`** — добавить JSDoc к пропу `closeOnSelection` (сейчас его
  нет), описывающий поведение и новый дефолт `false`.
- **`src/DateRangeInput.test.tsx`** — тест «closes the popover when a two-day
  range is completed» сейчас полагается на дефолт; теперь он должен явно
  передавать `closeOnSelection`. Добавляется тест: с дефолтом поповер остаётся
  открытым после выбора полного диапазона.
- **`docs/superpowers/specs/2026-05-20-date-range-input-design.md`** —
  синхронизировать §4 (пометка к `closeOnSelection` — `default false`).

## Обработка ошибок

Не требуется — это изменение значения по умолчанию.

## Тестирование (TDD)

- С дефолтом (без пропа): после выбора двух разных дат поповер остаётся
  открытым.
- С явным `closeOnSelection`: после выбора полного диапазона поповер
  закрывается.
- Существующие тесты (включая «keeps the popover open after the first day
  click») остаются зелёными.

## Объём

Одна крошечная фича, один план реализации. Публичный API не расширяется —
меняется только дефолтное значение существующего пропа `closeOnSelection`.
